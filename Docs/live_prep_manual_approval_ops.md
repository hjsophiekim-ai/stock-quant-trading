# Live Prep (Manual Approval) 운영 가이드

이 문서는 **실거래 자동 제출(무인 실행)을 허용하지 않는** 전제에서, 내일부터 사용할 수 있는 `live_manual_approval` 운용 절차를 정의합니다.

## 핵심 원칙

- 시스템은 `final_betting_v1`로 **후보를 생성**할 수 있지만, **실주문 제출은 항상 사용자 승인**이 필요합니다.
- 고빈도(분봉 단타) 전략은 `live_shadow`로만 운용하고, 실주문 제출은 금지(또는 건별 수동 승인 전용)합니다.
- 후보는 “추천”이 아니라 **시스템 후보(system candidates)** 입니다.
- `Emergency Stop`이 켜져 있으면 어떤 실주문도 제출되지 않습니다.

## 실행 모드

- `TRADING_MODE=live`
- `EXECUTION_MODE=live_manual_approval` (후보 생성 + 승인 후 제출 가능)
- `EXECUTION_MODE=live_shadow` (후보/신호 산출만, 제출 불가)

## 내일 운용 절차(권장)

1) 사전 체크
- 데스크톱/모바일에서 `Live Trading` 화면의 잠금 상태가 해제 가능한지 확인합니다.
- 필요 시 `Emergency Stop`을 ON으로 두고 점검합니다.

2) 후보 생성(final_betting_v1)
- 데스크톱 `국내 Live Trading` 화면에서 `final_betting_v1 후보 생성`을 실행합니다.
- 생성된 후보를 “시스템 후보”로 검토합니다(점수/근거/리스크 플래그/수량/손절%).

3) 승인 → 제출(원클릭)
- 후보별로 `Approve` 후 `Approve+Submit`을 눌러 제출합니다.
- 한 후보는 한 번만 제출됩니다(중복 제출 가드).

4) 비상 중단
- 이상 징후(주문 반복, 데이터 이상, 손실 급증)가 보이면 즉시 `Emergency Stop`을 ON으로 전환합니다.

## 제한/미검증 항목

- KIS 실계좌/실호스트에 대한 E2E 검증(네트워크/권한/시장 시간)은 환경 의존이라 자동 테스트로 보장하지 않습니다.
- `final_betting_v1`의 “오전 자동 청산” 같은 **자동 매도 제출**은 무인 실주문에 해당하므로 지원하지 않습니다.
- 손절/청산 자동화가 필요하면, “진입 승인 시 함께 제출되는 보호 주문(브로커 기능)” 중심으로 설계해야 합니다(브로커/주문 유형 지원 여부 확인 필요).

## 롤백

- `EXECUTION_MODE=live_shadow`로 내리면 제출 경로가 차단됩니다.
- `Emergency Stop=ON`으로 즉시 제출을 차단할 수 있습니다.
