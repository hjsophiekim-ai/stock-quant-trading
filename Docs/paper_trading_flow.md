# 모의투자 실행 흐름 (Paper Trading Flow)

## 문서 목적

실전 전에 모의투자로 전략과 리스크 제어를 검증하는 실행 절차를 설명합니다.

## 기본 원칙

- 기본 거래 모드는 `paper`
- 실거래 라우트는 잠금
- 리스크 규칙 통과 전 주문 금지

## 일일 실행 순서

1. 시작 전 점검
   - API 토큰 상태 확인
   - 리스크 한도 설정 확인
   - 전일 포지션/손익 동기화
2. 시장 데이터 수집
   - 대상 종목 시세/지표 갱신
3. 전략 신호 생성
   - 우량주 필터 통과 종목 선별
   - 추세 필터 적용
   - 분할매수 계획 수립
4. 리스크 엔진 검증
   - 손절 값 존재 여부
   - 일일 손실 제한 위반 여부
   - 총 손실 제한 위반 여부
5. 모의주문 실행
   - 주문 전송
   - 체결 확인 및 포지션 반영
6. 모니터링/기록
   - 주문/체결/손익 로그 저장
   - 경고 이벤트 발생 시 알림
7. 종료 후 리포트
   - 일일 성과 요약
   - 규칙 위반/개선 포인트 정리

## 중단 조건

- 일일 손실 제한 초과
- 총 손실 제한 초과
- 데이터/API 이상으로 판단 신뢰도 저하

중단 시에는 신규 주문을 막고 원인 분석을 먼저 수행합니다.
