# 시스템 설계 개요 (System Design)

## 문서 목적

이 문서는 프로젝트 전체를 "한눈에" 이해하기 위한 안내서입니다.  
초보자 기준으로, 어떤 데이터가 들어오고 어떤 단계를 거쳐 주문이 나가는지 설명합니다.

## 설계 목표

- 목표: 수익률 극대화
- 제약: 손실 최소화(최우선)
- 기본 모드: 모의투자
- 실거래: 잠금 상태 유지

## 상위 흐름

1. 시장 데이터 수집
2. 신호 생성(우량주 스윙 + 추세 필터 + 분할매매)
3. 리스크 엔진 검증
4. 주문 실행(기본: 모의주문)
5. 체결/포지션/성과 기록
6. 손실 한도 위반 시 즉시 거래 중단

## 주요 컴포넌트

- Data Ingestion: 시세/기본정보 수집
- Strategy Engine: 진입/청산 신호 계산
- Risk Engine: 손절/일일 손실 제한/총 손실 제한 강제
- Order Router: 모의/실거래 API 라우팅
- Portfolio Tracker: 보유/평가손익/성과 추적
- Monitoring & Alert: 로그, 경고, 중단 알림

## 실패 대응(Fail Safe)

- API 응답 이상/지연 발생 시 주문 중단
- 리스크 데이터 누락 시 신규 진입 금지
- 일일 손실 제한 또는 총 손실 제한 초과 시 자동 정지

## 운영 원칙

- "신호가 좋아도 리스크 조건 미통과면 주문 금지"
- "설명 가능한 결정"을 위해 모든 판단 근거 로그화
- 실거래 모드는 별도 승인 전까지 비활성
