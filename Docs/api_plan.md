# API 연동 계획 (KIS API Plan)

## 문서 목적

한국투자증권 API 연동을 안전한 단계로 진행하기 위한 계획서입니다.  
아래 순서는 반드시 지킵니다.

## 연동 순서(고정)

1. 토큰 발급
2. 조회 API
3. 모의주문
4. 실거래

## 1단계: 토큰 발급 (구현됨)

- 목표: 인증 성공, 토큰 만료/갱신 처리
- 체크포인트:
  - 키/시크릿은 환경변수로만 주입
  - 토큰 캐시 및 만료 시 재발급 처리
  - 실패 시 재시도 횟수 제한
  - KIS 응답 `rt_cd/msg1` 비즈니스 오류 처리

### 현재 코드 위치

- `app/auth/kis_auth.py`
  - `KISTokenAPI.request_access_token()`
  - `KISTokenService.get_valid_access_token()`

## 2단계: 조회 API (우선 구현)

- 목표: 시세/계좌 조회 안정성 확보
- 체크포인트:
  - 요청 제한(Rate Limit) 대응
  - 응답 스키마 검증(필수 필드 확인)
  - 장중/장후 데이터 차이 처리
  - 인증 헤더/Bearer 토큰 누락 방지

### 우선 대상 API

1. 잔고 조회 (`get_balance`)
2. 현재가 조회 (`get_quote`)
3. 보유종목 조회 (`get_positions`, 현재는 잔고 API `output1` 재사용)

### 현재 코드 위치

- `app/clients/kis_client.py`
  - 공통 래퍼: `_request`, `_get`, `_post`
  - 헤더: `_build_headers`
  - 조회 메서드: `get_balance`, `get_quote`, `get_positions`
  - 에러 처리: HTTP 오류 + KIS `rt_cd/msg1` 오류

## 3단계: 모의주문

- 목표: 주문 생성-전송-체결 확인까지 전 과정 검증
- 체크포인트:
  - 주문 전 리스크 엔진 승인 필수
  - 주문 요청/응답/체결 로그 저장
  - 오류 코드별 재시도/중단 정책

## 4단계: 실거래(잠금 해제 후)

- 기본은 잠금 상태로 유지
- 현재 정책: `LIVE_TRADING=true` 전에는 실주문 금지
- 잠금 해제 전 필수 조건:
  - 모의투자에서 일정 기간 안정성 확인
  - 손실 제한 규칙 검증 완료
  - 긴급 중단(킬 스위치) 동작 확인

## 공통 보안/운영 원칙

- API 키/토큰 저장소 커밋 금지
- 모든 주문 이벤트 감사 로그 기록
- 장애 발생 시 보수적으로 거래 중단

## 단계별 테스트 순서 (권장)

1. 토큰 발급 단위 테스트
2. 현재가 조회 연결 테스트
3. 잔고/보유종목 조회 테스트
4. 조회 결과를 전략 입력 스키마에 매핑
5. 모의주문 연동
6. 실거래 전환 체크리스트 검증
