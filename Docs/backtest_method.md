# Backtest Validation Method

## 목적

이 문서는 전략 평가를 "수익률 1개 숫자"가 아니라, 재현성과 안정성 중심으로 검증하기 위한 방법을 정의합니다.

> **중요 고지**  
> 월 15%는 연구 목표이며 보장 수익이 아님.
> 월 15% 목표는 연구용 목표이며 보장 수익이 아닙니다.

## 평가 원칙

- 단일 기간 백테스트 수익률보다 재현성과 리스크 지표를 우선
- 시장 국면별(상승/하락/횡보/고변동성) 성과를 분리 평가
- 파라미터 탐색은 반드시 워크포워드 검증으로 확인

## 핵심 지표

- 총수익률(Total Return)
- 월별 수익률(Monthly Return)
- 최대낙폭(Max Drawdown)
- 승률(Win Rate)
- 손익비(Payoff Ratio)
- Profit Factor
- Regime별 성과(Regime Performance)

## 모듈 구조

- `app/backtest/engine.py`
  - 거래 기록 기반 백테스트 실행
  - 수수료/슬리피지 반영
  - 에쿼티 커브 생성
- `app/backtest/metrics.py`
  - 총수익률/월별수익률/최대낙폭/승률/손익비/Profit Factor 계산
  - 국면별 성과 집계
- `app/backtest/walkforward.py`
  - 시간 분할(train/test) 자동 생성
  - train 구간 파라미터 선택 후 test 구간 검증
  - fold별 결과와 평균 요약 제공

## 워크포워드 검증 절차

1. 전체 기간을 train/test 구간으로 시간순 분할
2. train에서만 파라미터 후보 탐색
3. best 파라미터를 test 구간에 적용
4. 모든 fold에서 반복
5. fold 평균 성과와 편차를 함께 평가

## 과최적화 방지 체크

- 단일 구간 최고 성과보다 fold 일관성 확인
- 파라미터가 fold마다 극단적으로 바뀌는지 점검
- 거래 수가 너무 적은 "우연 성과" 모델 제외
- 최대낙폭과 rolling 손실 구간이 허용 범위 내인지 확인
- 하락장/고변동성 구간에서 손실 통제 여부 확인

## 실거래 전 최소 검증 체크리스트

- [ ] 워크포워드 전체 fold에서 극단적 붕괴 구간 없음
- [ ] 최대낙폭이 사전 정의 한도 이내
- [ ] 하락장/고변동성 구간에서 신규 진입 통제가 동작
- [ ] Profit Factor와 손익비가 과도한 한두 거래에 의존하지 않음
- [ ] 거래 비용(수수료/슬리피지) 반영 후에도 성과 유지
- [ ] 리스크 엔진(일일/총손실/rolling loss/cooldown) 연동 검증 완료
- [ ] 모의투자에서 동일 로직 재현 확인
- [ ] 실거래 잠금 해제 전 LIVE 체크리스트(플래그/승인/경고 배너) 재확인
