# 사용자 Quick Start (Windows 데스크톱)

이 문서는 **처음 사용하는 일반 사용자**를 위한 안내입니다.  
목표는 간단합니다: **설치 -> 로그인 -> 대시보드 확인 -> 모의투자(Paper) 실행**.

> 중요: 이 제품은 현재 **live trading 잠금 상태**가 기본입니다.  
> 안내를 따라도 실거래가 자동으로 켜지지 않습니다.

---

## 0) 시작 전에 확인

- Windows 10/11
- 인터넷 연결
- 운영자가 전달한 설치 파일: `Stock Quant Desktop-Setup-...exe`
- 운영자가 전달한 서버 상태(정상 운영 중)

---

## 1) 설치 후 실행 방법

1. 설치 파일(`.exe`)을 더블 클릭합니다.
2. 설치 마법사에서 `다음`을 눌러 설치를 완료합니다.
3. 바탕화면의 **Stock Quant Desktop** 아이콘을 실행합니다.
4. 앱이 열리면 로그인 화면이 보입니다.

---

## 2) 로그인 방법

1. 이미 계정이 있으면 이메일/비밀번호 입력 후 **로그인**.
2. 처음이면 **회원가입(첫 실행)** 버튼으로 계정 생성 후 로그인.
3. 로그인 성공 시 자동으로 **대시보드**로 이동합니다.

팁:
- 로그인 유지 옵션을 켜면 다음 실행 때 자동 로그인 시도합니다.

---

## 3) 브로커 계정 연결 방법

대시보드에서 **브로커 설정(Broker Settings)** 으로 이동합니다.

입력 항목:
- KIS App Key
- KIS App Secret
- 계좌번호(CANO)
- 계좌상품코드(예: `01`)
- trading mode: **paper** 선택

입력 후 **저장**을 누릅니다.

---

## 4) Connection Test 방법

브로커 설정 화면에서 **연결 테스트** 버튼을 누릅니다.

성공 기준:
- 상태가 `success` 또는 `ok=true`로 표시
- 마지막 테스트 시각이 갱신

실패하면 아래 "자주 발생하는 오류"를 참고하세요.

---

## 5) Paper Trading 시작 방법

1. 하단/사이드 메뉴에서 **Paper Trading** 화면으로 이동
2. 전략 선택 (`swing_v1`, `bull_focus_v1`, `defensive_v1`)
3. **시작** 버튼 클릭

주의:
- 브로커가 `paper` 모드가 아니거나 연결 테스트가 실패면 시작 버튼이 비활성화되거나 시작이 거부됩니다.
- `risk_off` 상태가 되면 원인 확인 후 `risk 해제` 또는 `중지 후 재시작`이 필요합니다.

---

## 6) 결과/수익률 보는 방법

- **Paper Trading 화면**: 상태, 최근 로그, 포지션, 틱 기준 손익 요약 확인
- **대시보드**: 전체 요약 확인
- **성과(Performance)**: 수익률/포지션/거래 내역 확인

---

## 7) 자주 발생하는 오류와 해결법

### A. 로그인 실패
- 원인: 이메일/비밀번호 오입력, 계정 미생성
- 해결: 비밀번호 재확인, 먼저 회원가입

### B. 서버 연결 실패
- 원인: 서버 점검 중, 네트워크 문제
- 해결: 잠시 후 재시도, 운영자에게 서버 상태 문의

### C. 브로커 연결 테스트 실패
- 원인: App Key/Secret 오입력, 계좌번호 형식 오류, KIS 점검 시간
- 해결: 입력값 다시 확인, 잠시 후 재시도

### D. Paper 시작 불가(403/409 등)
- 원인:
  - 연결 테스트 미통과
  - trading mode가 paper 아님
  - 다른 세션이 이미 실행 중
- 해결:
  - 브로커 상태 success 확인
  - mode를 paper로 저장
  - 기존 세션 중지 후 재시도

### E. risk_off 상태
- 원인: 연속 오류 누적
- 해결: 로그 확인 -> 원인 수정 -> risk 해제 또는 중지 후 시작

---

## 8) 꼭 기억할 점

- 이 가이드는 **모의투자(paper)** 용입니다.
- **live trading은 잠금 상태**이며, 일반 사용자가 임의로 활성화할 수 없습니다.
