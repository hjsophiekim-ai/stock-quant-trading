# 사용자 Quick Start (Android)

이 문서는 **안드로이드 사용자**가 처음 앱을 설치하고  
바로 로그인/대시보드/모의투자를 확인할 수 있도록 작성했습니다.

> 중요: 현재 기본 정책은 **paper trading(모의투자)** 입니다.  
> **live trading은 잠금 상태**이며 자동으로 켜지지 않습니다.

---

## 0) 시작 전 준비

- Android 스마트폰
- 인터넷 연결
- 운영자가 전달한 앱 설치 파일(APK) 또는 스토어 설치 안내
- 운영 중인 서버(클라우드 backend) 연결 가능 상태

---

## 1) 설치 후 실행 방법

### APK로 설치한 경우
1. APK 파일을 실행해 설치합니다.
2. 보안 경고가 나오면 안내에 따라 설치 허용합니다.
3. 설치 완료 후 앱을 실행합니다.

### 스토어 설치한 경우
1. 앱 설치
2. 앱 실행

앱이 열리면 로그인 화면이 보입니다.

---

## 2) 로그인 방법

1. 계정이 있으면 이메일/비밀번호로 로그인
2. 처음이면 회원가입 후 로그인
3. 로그인 성공 시 **대시보드로 자동 이동**

팁:
- 로그인 유지를 켜면 다음 실행 때 자동 로그인 시도

---

## 3) 브로커 계정 연결 방법

하단 메뉴에서 **브로커**(Broker Settings) 화면으로 이동:

입력:
- KIS App Key
- KIS App Secret
- 계좌번호
- 계좌상품코드
- trading mode는 **paper**

저장 후 연결 테스트를 진행합니다.

---

## 4) Connection Test 방법

브로커 화면의 **연결 테스트** 버튼 실행

성공 기준:
- 상태가 성공(success/ok)으로 표시
- 마지막 테스트 시간이 갱신

---

## 5) Paper Trading 시작 방법

1. 하단 메뉴에서 **Paper**로 이동
2. 전략 선택
3. **시작** 버튼 탭

중지:
- 중지 버튼으로 언제든 종료 가능

risk_off:
- 로그 확인 후 `risk 해제` 또는 `중지 후 재시작`

---

## 6) 결과/수익률 보는 방법

- **대시보드**: 상태 요약
- **Paper 화면**: 로그, 포지션, 손익 요약
- **성과(Performance)**: 수익률·포지션·거래 내역

---

## 7) 자주 발생하는 오류와 해결법

### A. 네트워크 오류
- 원인: 인터넷 불안정, 서버 점검
- 해결: 네트워크 재연결, 잠시 후 재시도

### B. 로그인 실패
- 원인: 계정 정보 오입력
- 해결: 이메일/비밀번호 확인, 회원가입 여부 확인

### C. 브로커 테스트 실패
- 원인: 키/계좌 입력 오류, KIS 응답 지연
- 해결: 값 재입력, 잠시 후 테스트 재실행

### D. Paper 시작 안 됨
- 원인: 브로커 연결 실패, mode 미스매치, 세션 충돌
- 해결: 브로커 성공 상태 확인, mode=paper 저장, 기존 세션 중지

### E. 결과가 바로 안 갱신됨
- 원인: 루프 주기/틱 지연
- 해결: 10~30초 기다린 뒤 새로고침/재확인

---

## 8) 꼭 기억할 점

- 앱에는 KIS 키를 저장하지 않고, 서버에 암호화 저장합니다.
- 앱은 로그인 후 backend API만 사용합니다.
- **live trading은 잠금 상태**입니다.
