# AGENTS.md

이 문서는 이 프로젝트에서 작업하는 사람/AI 에이전트가 따라야 할 공통 규칙입니다.

## 1) 프로젝트 정체성

- 이 프로젝트는 **한국투자증권 Open API 기반 자동매매 시스템**입니다.
- 목표는 수익률 개선이지만, 최우선 원칙은 **손실 최소화**입니다.
- 목표 문장: **수익률 극대화를 추구하되 손실 최소화를 최우선 제약으로 둔다**
- **월 15%는 연구용 목표일 뿐 보장 목표가 아닙니다.**
- 모든 기능은 "수익 증가"보다 "생존 가능성"을 먼저 검증합니다.

## 2) 거래 모드 정책

- 기본 모드는 `paper trading` 입니다.
- 실거래는 기본 잠금 상태입니다.
- 환경변수 `LIVE_TRADING=true` 가 아니면 **실주문을 절대 실행하지 않습니다**.
- 추가 확인 플래그 `LIVE_TRADING_CONFIRM=true` 가 없으면 실주문을 차단합니다.
- 잠금 해제는 명시적 설정 + 체크리스트 통과 시에만 허용합니다.

## 3) 작업 우선순위

1. 리스크 엔진/안전장치
2. 데이터 정확성
3. 전략 로직
4. 자동화/성능

## 4) 전략 실행 기본 흐름

- 시장 국면 인식 엔진(`market_regime`)을 **반드시 먼저** 통과합니다.
- 국면 판별 후 전략 라우팅(`bull`/`bear`/`sideways`/`defensive`)을 수행합니다.
- 후보 종목은 `ranking engine`으로 점수화하여 상위 N개만 진입 검토합니다.

## 5) 필수 안전 체크

- 주문 전 리스크 검증(손절/일일 손실 제한/총 손실 제한)
- `rolling loss limit` 및 `trading cooldown` 상태 확인
- 시장 데이터 유효성 검증(결측/지연/이상치)
- API 오류 시 재시도 및 안전 중단(Fail Safe)
- 로그/알림으로 의사결정 근거 남기기
- 손절 규칙은 절대 우선이며, 예외를 두지 않습니다.
- 리스크 엔진의 판단은 전략 신호보다 항상 우선합니다.
- `bearish_trend`, `high_volatility_risk`에서는 **방어 로직 우선**으로 동작합니다.

## 6) 코딩 가이드(초보자 친화)

- 함수는 작고 테스트 가능하게 유지합니다.
- 변수명은 의미가 드러나게 작성합니다.
- "왜 필요한지" 중심의 짧은 주석을 허용합니다.
- 한 파일에 역할을 과도하게 섞지 않습니다.
- 핵심 로직을 수정하면 관련 `tests` 도 반드시 함께 수정합니다.
- 핵심 로직을 수정하면 `docs/` 도 반드시 함께 수정합니다.

## 7) 핵심 엔진 유지 원칙

- `trailing exit` 로직을 유지합니다.
- `dynamic sizing` 로직을 유지합니다.
- `drawdown control` 로직을 유지합니다.
- `ranking engine` 로직을 유지합니다.
- 성능 개선 시에도 위 4개 안전/품질 엔진을 훼손하지 않습니다.

## 8) 보안/문서/품질 원칙

- 비밀정보(키/토큰/계좌정보)는 `.env` 에서만 읽습니다.
- 비밀키/토큰은 절대 저장소에 커밋하지 않습니다.
- 코드 변경 시 `README.md` 와 `docs/` 를 함께 갱신합니다.
- 앱 사용자 인증은 JWT(access/refresh) 기반으로 구현하고, 비밀번호는 해시만 저장합니다.
- 사용자별 설정/브로커 계정/권한(role: `admin`/`user`)은 분리 가능한 구조로 유지합니다.
- 브로커 비밀정보 입력은 앱에서 받되, 장기 저장은 서버 암호화 저장소에서만 수행합니다.
- 한국투자 토큰 발급 테스트는 서버에서만 호출하며, 앱은 상태만 조회/표시합니다.
- 앱의 모의투자 실행/중지/모니터링 기능은 `paper-trading` 전용 API로 분리합니다.
- 모의투자 제어 API는 `PaperBroker`만 사용하며, live 주문 경로와 로직을 혼합하지 않습니다.
- 코어 실행기(`python -m app.main`)에서 KIS **모의투자 REST**와 주문을 연결할 때는 `KisPaperBroker`만 사용합니다(`https://openapivts...` 전용, 실전 도메인과 분리). 환경변수 `PAPER_USE_KIS_EXECUTION`, `PAPER_TRADING_LOOP` 등으로 켭니다.
- 백엔드 **앱 사용자 Paper 루프**(`UserPaperTradingLoop`)는 모의 OAuth를 주기마다 무조건 재발급하지 않고 잔고 probe로 유효성을 확인한 뒤 필요 시에만 재발급하며, 실패 시 in-memory·SQLite 캐시·백오프로 복구한다. 연속 OAuth 실패는 일반 틱 실패 streak과 분리해 `risk_off`한다(`PAPER_KIS_TOKEN_*`, `PAPER_SESSION_MAX_TOKEN_FAILURES_BEFORE_RISK_OFF`).
- live trading unlock은 환경변수 + 앱 다중 확인 + 추가 승인 상태를 모두 요구합니다.
- `LIVE_TRADING=true` 단독으로는 live 주문을 허용하지 않습니다.
- live 안전 설정 변경은 반드시 이력(history)으로 남기고, 앱에 위험 경고를 표시합니다.
- 배포 설정에서 모바일/데스크톱에는 비밀정보를 넣지 않고 서버 URL 등 비민감 값만 주입합니다.
- 실거래 관련 코드는 잠금 플래그 없이 동작하지 않도록 보호합니다.
- live trading 관련 코드는 일반 코드보다 더 엄격한 보호 규칙을 적용합니다.
  - 다중 플래그 확인
  - dry-run 로그
  - startup safety validation
  - 손실 제한 초과 시 자동 shutdown 경로 유지

## 9) API 접근 경계

- `app/auth` 와 `app/clients` 만 한국투자증권 API에 직접 접근할 수 있습니다.
- 나머지 모듈(`strategy`, `risk`, `orders`, `portfolio`, `scheduler`)은 API 직접 호출을 금지합니다.
- 외부 API 연동은 클라이언트 계층을 통해서만 수행합니다.

## 10) 작업 환경 규칙

- Codex는 반드시 **현재 로컬 디렉터리**에서만 작업합니다.
- cloud workspace 경로 및 원격 작업 경로 사용을 금지합니다.

## 11) 작업 종료 후 동기화 규칙

- 모든 작업 완료 후 `AGENTS.md`가 현재 프로젝트 상태와 규칙을 반영하는지 확인합니다.
- 규칙/구조/안전 정책이 바뀌었으면 `AGENTS.md`를 즉시 업데이트합니다.
- 업데이트 시 핵심 변경 사유를 간단히 남기고, 최신 전체 내용을 공유합니다.
- 앱/서버 API 스키마가 바뀌면 `docs/api_contract_app.md`와 `shared/models`를 함께 동기화합니다.

## 12) 한국투자증권 연동 순서(준수 필수)

1. 토큰 발급
2. 조회 API 검증
3. 모의주문 검증
4. 실거래 전환(잠금 해제)

순서를 건너뛰는 구현은 허용하지 않습니다.

- 로컬에서 모의 계좌·백엔드까지 빠르게 띄우는 절차는 `docs/quickstart_real_mock_trading.md` 와 `env.paper.example` / `scripts/run_backend.*` 를 따른다.
- 일반 사용자 설치형 제품(Win 설치본·Android·백엔드 번들) 목표는 `docs/product_architecture.md` 에 정리한다.
